# 2.2 面试流程

> 原文：[`huyenchip.com/ml-interviews-book/contents/2.2-interview-pipeline.html`](https://huyenchip.com/ml-interviews-book/contents/2.2-interview-pipeline.html)

机器学习角色的面试流程是从经过验证的软件工程面试流程演变而来的，并包括在传统技术面试过程中可以看到的相同组件。

有很多人参与面试过程。对于招聘经理来说，为每个面试官分配一套要评估的技能至关重要，这样不同的面试官会问不同的问题，并且他们可以集体获得对你所有关心技能的整体了解。一个面试官可能会问你关于理论的问题，几个关于编码的问题，一个关于机器学习系统设计的问题。理想情况下，面试官会告诉你他们想要关注的技能，这样你就可以调整你的回答来突出这些技能。如果他们没有这样做，你可以询问。

经常鼓励招聘人员与候选人分享面试官的名字。如果他们不分享，你可以询问。你可以在面试前查找你的面试官，了解他们的工作。这将不仅让你对加入公司后将要做什么有一个感觉，还能了解你的面试官的兴趣领域。

你还应该询问你被考虑加入的团队。在大多数公司，你面试的是特定的团队。然而，如果你申请的是谷歌和 Facebook 这样的公司，你会在通过后与团队匹配。这意味着，如果没有团队接受你，你可能会在面试过程中通过而没有得到工作机会，尽管这种情况很少见^(40)。

以下面试组件的列表很长且令人畏惧，但公司通常只使用其中的一部分。每个组件中的面试数量也各不相同。如果公司对你能力的信心十足，他们可能会跳过任何步骤。优秀的候选人甚至可能在没有经过所有先前步骤的情况下直接被邀请到现场。另一方面，需要为现场面试而旅行的候选人可能在面试前受到更严格的审查。

整个过程可能会很长且很累人，但这对每个候选人来说都是如此。你不必完美地回答所有问题。你只需要比其他候选人做得更好。没有公司期望你什么都知道。这个领域发展得如此之快，期望任何候选人知道所有最新的论文和技术是不现实的。
