# 1.1.3.2 机器学习工程师与软件工程师的比较

> 原文：[`huyenchip.com/ml-interviews-book/contents/1.1.3.2-machine-learning-engineer-vs.-software-engineer.html`](https://huyenchip.com/ml-interviews-book/contents/1.1.3.2-machine-learning-engineer-vs.-software-engineer.html)

机器学习工程被认为是软件工程的子领域。在大多数组织中，机器学习工程师（MLE）的招聘流程是从他们现有的软件工程师（SWE）招聘流程中分离出来的。一些组织可能会用一些针对机器学习的特定问题替换掉几个 SWE 问题。一些组织只是在现有的 SWE 面试流程上增加一个专门针对机器学习的面试，这使得他们的 MLE 流程比 SWE 流程要长一些。

总体而言，机器学习工程师候选人应了解如何编码，并熟悉软件工程工具。许多传统的 SWE 工具可以用来开发和部署机器学习应用。

在机器学习早期应用阶段，当公司对机器学习生产过程了解不多时，许多公司曾期望机器学习工程师（MLE）候选人既是杰出的软件工程师，又是杰出的机器学习研究者。然而，找到符合这种描述的候选人证明是困难的，许多公司已经放宽了他们的机器学习标准。实际上，几位招聘经理告诉我，他们更愿意雇佣那些是优秀工程师但对机器学习了解不多的人，因为优秀的工程师学习机器学习比机器学习专家学习良好的工程实践要容易得多。

> 🌳 **提示** 🌳
> 
> 如果你是一名正在考虑软件工程和机器学习之间选择的候选人，选择工程。
