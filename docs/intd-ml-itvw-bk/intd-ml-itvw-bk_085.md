# 6.3.1 数据结构

> 原文：[`huyenchip.com/ml-interviews-book/contents/6.3.1-data-structures.html`](https://huyenchip.com/ml-interviews-book/contents/6.3.1-data-structures.html)

如果我们的世界是由数据驱动的，那么数据结构就是阻止我们陷入混乱的东西。从数字时代的黎明开始，计算机科学中最优秀的大脑就夜以继日地思考着如何高效地存储和处理数据。在机器学习中，数据结构的重要性更是不言而喻，因为该领域是由大数据推动的。

虽然有一些经典的数据结构经受了时间的考验，但随着新格式的引入和新数据的产生，开发新的数据结构和改进现有数据结构永远是一场没有终点的战斗。你对现有数据结构的熟悉程度、对它们实现方式的理解，以及对何时使用何种数据结构的直觉将非常有价值。

你应该了解一些运行时间复杂度，并且至少能在一种语言中实现它们的数据结构：

+   树：二叉搜索树、堆、前缀树（前缀和后缀树）

+   队列、栈、优先队列

+   链表

+   HashMap 和 HashTable

我们在这里没有关于这些数据结构的问题，但你应该尝试自己实现它们，无论是在编码练习网站上还是在本地实现，并与已知的实现进行比较。

你应该熟悉操作流行的数据格式，例如无处不在的 CSV 格式和适合网络和序列化的 JSON 格式。CSV 和 JSON 都是传统基于行的文件格式的例子：数据是按行存储和索引的。

近年来，基于列的格式变得越来越普遍，因为它允许大数据应用通过调用对应于该特征的一列来快速从所有数据点中提取一个特征。流行的机器学习数据框架，如`pandas`和`dask`，针对基于列的操作进行了优化。两种常见的基于列的文件格式是 Apache Hadoop 支持的 Parquet 和 Apache Hive 支持的 ORC。

基于行的数据格式在写入时更高效，而基于列的格式在读取时更高效。如果你的数据是一次写入多次读取，则使用基于列的格式。如果需要定期重写，则选择基于行的格式。
