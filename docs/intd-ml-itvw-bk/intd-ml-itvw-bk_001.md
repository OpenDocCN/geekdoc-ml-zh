# 机器学习面试入门书

> 原文：[`huyenchip.com/ml-interviews-book/`](https://huyenchip.com/ml-interviews-book/)

你可以在[这里](https://huyenchip.com/ml-interviews-book/)阅读本书的网页友好版本。你可以在[GitHub](https://github.com/chiphuyen/ml-interviews-book)上找到源代码。讨论书中问题答案的 Discord 群组[在这里](https://discord.gg/XjDNDSEYjh)。

作为候选人，我在十几家大公司和初创公司进行了面试。我收到了包括谷歌、英伟达、Snap、Netflix、Primer AI 和 Snorkel AI 在内的公司在机器学习职位上的工作机会。我也在其他许多公司遭到了拒绝。

作为面试官，我参与了英伟达和 Snorkel AI 的招聘流程的设计和执行，从向那些我热爱其工作的候选人发送冷邮件，筛选简历，进行探索性和技术面试，讨论是否要雇佣候选人，到试图说服候选人选择我们而不是竞争性报价。

作为朋友和老师，我帮助许多朋友和学生为大公司和初创公司准备机器学习面试。我给他们进行模拟面试，并记录他们经历的过程以及他们被问到的问题。

我还咨询了几家初创公司关于他们的机器学习招聘流程。在没有强大的内部机器学习团队和流程来帮助你评估候选人的情况下，招聘机器学习职位变得相当困难。由于机器学习在行业中的应用仍然相对较新，许多公司仍在边走边摸索，这并没有让候选人更容易。

这本书是许多人在面试桌的两侧坐下并花费大量时间思考招聘过程的结果。它是以候选人为出发点编写的，但看到早期草稿的招聘经理告诉我，他们发现了解其他公司的招聘方式以及重新思考自己的流程很有帮助。

本书分为两部分。第一部分概述了机器学习面试流程，包括哪些类型的机器学习职位可用，每个职位需要哪些技能，常见的问题类型以及如何准备这些问题。这部分还解释了面试官的心态以及他们寻找的信号。

第二部分包含超过 200 个知识问题，每个问题都标注了其难度级别——对于更高层次的职位面试应预期更难的问题——涵盖了机器学习中的重要概念和常见误解。

在你完成这本书之后，你可能想查看[30 个开放性问题](https://huyenchip.com/machine-learning-systems-design/toc.html)来测试你将所学知识组合起来解决实际挑战的能力。这些问题测试了你的问题解决技能以及你在实施和部署机器学习模型方面的经验范围。一些公司称它们为**机器学习系统设计**问题。我交谈过的几乎所有公司都在他们的面试过程中至少问过这类问题，而且这些通常是候选人认为最困难的问题。

“机器学习系统设计”是一个复杂的话题，值得有一本自己的书。要了解更多关于它的信息，请查看我在斯坦福的[**CS 329S：机器学习系统设计**](https://stanford-cs329s.github.io/)课程。

本书不是机器学习教科书的替代品，也不是面试的捷径。它是一个工具，用于巩固你在机器学习方面的现有理论和实践知识。本书中的问题也有助于识别你的盲点/弱点。每个主题都附有资源，可以帮助你加强对该主题的理解。
