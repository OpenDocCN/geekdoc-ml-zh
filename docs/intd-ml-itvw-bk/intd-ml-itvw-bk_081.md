# 第六章 计算机科学

> 原文：[`huyenchip.com/ml-interviews-book/contents/chapter-6.-computer-science.html`](https://huyenchip.com/ml-interviews-book/contents/chapter-6.-computer-science.html)

对于编码问题，最好的准备方法是每天写代码。编程精通需要持续的练习。

然而，你每天做的编码类型与面试中要求做的编码类型不同，所以进行一些练习是个好主意。在面试之前，做一些典型的软件工程问题，让自己进入问题解决、白板编码模式。

如果你有一段时间没有看过它们了，就找一本关于算法和数据结构的好书浏览一下。我们推荐 Michael T. Goodrich 所著的《Python 中的数据结构与算法》以及托马斯·科门等人所著的经典《算法导论》。

我们还推荐练习 LeetCode、CodeSignal 和 HackerRank 等网站。这些网站按难度排名问题——你应该尝试解决中等和困难的问题。大多数网站都有解决方案可供参考，以防你想比较你的解决方案与更优的解决方案^(1)。

本章讨论了机器学习面试中涵盖的计算机科学的三个主要方面：算法、复杂性和数值分析，以及数据，包括数据结构。

> 🌳 **面试时使用哪种编程语言** 🌳
> 
> 如果你只熟悉一种语言，在面试时使用它时不必感到愧疚。如果你熟悉多种语言，在选择语言之前先听一下问题。这表明你理解不同的语言是为不同的目的而构建的：适合一项任务的语言可能不适合另一项任务。
> 
> 在你的简历上列出你所知道的所有语言，但在面试时不必炫耀。雇主更愿意雇佣一个精通一种语言的人——这意味着你可以学会精通其他语言——而不是雇佣一个在多种语言上都平庸的人。
> 
> 根据你选择的语言，面试官可能会推断出你对什么感兴趣。例如，一位招聘经理告诉我，如果一个候选人选择用 Python 实现数据结构，他知道这个候选人并不关注性能。
> 
> Python 已经成为机器学习的默认通用语言——大多数框架都有 Python API，大多数开源项目都是用 Python 编写的。了解这门语言是有用的，而且大多数面试官可能都期望你使用它，但不必觉得在面试时必须使用它。如果你更熟悉另一种语言，就使用它。用另一种语言，比如 Swift，编写一个复杂的模型会给人留下更深刻的印象，并帮助你脱颖而出。
> 
> 如果你至少掌握一门以性能为导向的语言，例如 C++ 或 Go，那也会有所帮助。C++ 由于拥有更多的支持而更为流行，但 Go 更易于学习和管理。随着越来越多的机器学习模型被作为网络应用提供服务，许多初创公司都在寻找具备前端技能的机器学习工程师。精通 TypeScript 或 React 将是一个巨大的加分项。
