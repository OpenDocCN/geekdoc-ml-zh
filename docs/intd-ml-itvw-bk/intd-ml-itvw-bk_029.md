# 2.1.1.2 非技术技能

> 原文：[`huyenchip.com/ml-interviews-book/contents/2.1.1.2-non-technical-skills.html`](https://huyenchip.com/ml-interviews-book/contents/2.1.1.2-non-technical-skills.html)

1.  **分析思维**，或有效解决问题的能力。这涉及到将复杂问题逐步分解为可管理的组件的方法。你可能一开始不知道如何解决问题，尤其是如果你从未遇到过这种情况，但你应该知道如何系统地接近它。在招聘初级职位时，雇主可能比其他任何技能都更重视这项技能。你可以用几周的时间教某人 Python，但教人如何思考可能需要数年。

1.  **沟通技巧**。现实世界的机器学习项目涉及来自不同背景的许多不同利益相关者：机器学习工程师、DevOps 工程师、领域专家（例如医生、银行家、律师）、产品经理、商业领袖。向那些也参与开发过程但可能没有技术背景的人解释你的机器学习模型的技术方面非常重要。

    与无法解释他们所做事情的人合作是困难的。如果你有一个绝妙的主意，但没有人能理解它，那么它就不是绝妙的。记住，基本复杂的思想与作者无法清楚表达而变得复杂的思想之间存在着巨大的差异。

1.  **经验**。你过去是否完成过类似任务，以及你是否能够从这些经验中归纳出对未来任务的见解。科技行业因贬低经验而著称，更倾向于雇佣那些即将燃尽精力的二十多岁年轻人。然而，在机器学习领域，由于改进通常基于经验观察，经验在模型在基准数据集上表现良好和在真实世界数据上实时工作之间起着决定性作用。经验与资历不同。有些工程师虽然工作了几十年，但他们的经验可能还不及一个好奇的大学生。

1.  **领导力**。在这个语境中，领导力意味着能够主动发起并完成任务的能力。如果你被分配了一项任务，你是否能够从头到尾独立完成，而不需要别人的指导？你不需要知道如何独自完成所有组件，但你应该知道你需要什么样的帮助，并且应该主动寻求。这种品质可以通过你过去的项目来评估。在学校或你之前的工作中，你是否只是按照指示行事，或者你是否抓住了机会并主动采取行动？

所需的技能集因角色而异。请参阅**1.1 不同的机器学习角色**部分了解角色之间的差异。
